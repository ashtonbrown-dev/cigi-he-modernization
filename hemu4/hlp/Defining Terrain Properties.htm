CIGI Host Emulator 3

Defining Terrain Properties

The TERRAIN.DEF file is used to configure terrain database parameters for the CIGI Host Emulator. This configuration includes a database number matching that used by the IG to identify each database, the origin of each database, default Ownship positions, and definitions for database components.

The following is an example terrain database definition:

database
{
   name = "Nellis";
   id = 3;
   
   // The following parameters define the
   // origin of the database:
   origin_latitude  = 36.4995;
   origin_longitude = -114.9935;
   origin_altitude  = 0.0;
   
   // The following parameters define the
   // default ownship position within the
   // database:
   default_latitude  = 36.5;
   default_longitude = -115.0;
   default_altitude = 3000.0;
   
   // The following definition specifies
   // the name, ID, and valid states for a
   // discrete component contained within
   // the database.
   component
   {
      name = "Runway Lights";
      id = 5;
      instance_id = 0;
      def_state = 1;
      def_data1 = 100, INT32;
      
      state
      {
         name = "Off";
         value = 0;
      }
      
      state
      {
         name = "On";
         value = 1;
      }
   }
}

DATABASE definitions define the following parameters:

  • NAME
    A descriptive, user-recognizable name for the database. The name must be enclosed within double quotation marks ("…").
  • ID
    An integer from 1 through 127 that specifies the database.
  • ORIGIN_LATITUDE
    A floating-point number that specifies the geodetic latitude of the origin point of the database.
  • ORIGIN_LONGITUDE
    A floating-point number that specifies the geodetic longitude of the origin point of the database.
  • ORIGIN_ALTITUDE
    A floating-point number that specifies the altitude in meters of the origin point of the database. The altitude is with respect to Mean Sea Level.
  • DEFAULT_LATITUDE
    A floating-point number that specifies the default latitude of the Ownship when it is created or when the database is changed. If this parameter is not specified, ORIGIN_LAT is used.
  • DEFAULT_LONGITUDE
    A floating-point number that specifies the default longitude of the Ownship when it is created or when the database is changed. If this parameter is not specified, ORIGIN_LON is used.
  • DEFAULT_ALTITUDE
    A floating-point number that specifies the default altitude of the Ownship when it is created or when the database is changed. If this parameter is not specified, ORIGIN_ALT is used.
  • COMPONENT
    Defines a component for entities of the given type.  Discrete component definitions specify the following parameters:
    • NAME
      A descriptive, user-recognizable name for the component. The name must be enclosed within double quotation marks ("…").
    • ID
      An integer that uniquely identifies the component for a given entity.
    • INSTANCE_ID

      An application- or database-specific object instance identifier.  See the CIGI 3 ICD for more information.

    • DEF_STATE
      An integer that specifies the default discrete state for the component.
    • DEF_DATA1
      An optional pair of parameters that specify the default value and datum type for the 1st user-defined datum of the component. The value may be an integer or floating-point value. The datum type is specified by one of the following keywords:

       
      Keyword Description
      INT32 32-bit signed integer
      UINT32 32-bit unsigned integer
      INT64 64-bit signed integer
      UINT64 64-bit unsigned integer
      FLOAT 32-bit single-precision floating-point number
      DOUBLE 64-bit double-precision floating-point number

      The value must precede the datum type and the two must be separated by a comma.
    • DEF_DATA2
      An optional pair of parameters that specify the default value and datum type for the 2nd user-defined datum of the component. The parameters are used as described by DEF_DATA1. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA3
      An optional pair of parameters that specify the default value and datum type for the 3rd user-defined datum of the component. The parameters are used as described by DEF_DATA2. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA4
      An optional pair of parameters that specify the default value and datum type for the 4th user-defined datum of the component. The parameters are used as described by DEF_DATA3. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA5
      An optional pair of parameters that specify the default value and datum type for the 5th user-defined datum of the component. The parameters are used as described by DEF_DATA4. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA6
      An optional pair of parameters that specify the default value and datum type for the 6th user-defined datum of the component. The parameters are used as described by DEF_DATA1; however, the datum type must be a 32-bit type. This parameter pair is not used if DEF_DATA5 specifies a 64-bit data type.
    • STATE
      Defines a discrete state for the component. Each STATE block must include the following parameters:
      • NAME
        A descriptive, user-recognizable name for the component state. The name must be enclosed within double quotation marks ("…").
      • VALUE
        An integer that specifies the value associated with the state.

      The parameters of the STATE block must be contained within a single set of brackets ({...}).
