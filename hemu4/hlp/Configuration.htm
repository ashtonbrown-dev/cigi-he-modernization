CIGI Host Emulator 3

Configuration

The CIGI Host Emulator uses ASCII text files to define entity types and default view, environmental, and database properties. When the program starts, it loads the configuration from the definition (*.DEF) files located in the "Default" subfolder of the Host Emulator program folder. See Loading a New Configuration for more information.

The general format for definition files is as follows:

BLOCK_NAME
{
    PARAMETER = VALUE;
    ...
    BLOCK_NAME
    {
        PARAMETER = VALUE;
        ...
        BLOCK_NAME
        {
            PARAMETER = VALUE;
            ...
        }
    }
}

BLOCK_NAME describes the type of definition block that follows. This should be replaced by one of the definition keywords appropriate to the specific definition file (see the appropriate topic in this section). In some cases, definition blocks may be nested. For example, the ENTITIES.DEF file contains ENTITY definitions, which may contain one or more DISCRETE_COMPONENT definitions, which must contain STATE definitions.

PARAMETER can be the name of any configuration parameter that is applicable to the definition block.

Keep the following rules in mind when creating or editing definition files:

  • Keywords are not case-sensitive.
  • Definition blocks must be enclosed in braces ({…}).
  • An equals sign (=) must separate each parameter name from its assigned value.
  • A semicolon (;) must follow each value, regardless of its type.
  • Character string values must be enclosed in double quotation marks ("…") and must be on a single line.
  • Whitespaces (i.e. spaces, tabs, and carriage-returns) are ignored unless contained within quotation marks.
  • Comments can either be C-style (/*…*/) or C++-style (//…) comments.
