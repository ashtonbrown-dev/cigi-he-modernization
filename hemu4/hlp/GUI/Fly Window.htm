Fly Window

The Fly window is used to control the motion of an entity using a joystick or a simulated joystick interface. This window can be activated by clicking the Fly button on the Entity Tree toolbar. The Fly window is shown below.

The top-left portion of the window is occupied by a simulated joystick. Dragging the ball with the mouse will simulate the manipulation of a joystick peripheral. If an actual joystick is connected and configured on the PC, the ball will move in response to joystick motion. For more information on using the joystick, see Using a Joystick Peripheral.

Along the bottom and right sides of the simulated joystick window are two sliders. These will move the ball along either the up/down or left/right axis. When these sliders are released, the ball will stay "locked" in its current position. Clicking or dragging the ball will release the lock and allow the ball to snap back to center.

In the bottom-left portion of the window are two drop-down lists. The first of these contains all of the child entities that represent missiles. The list below it is a list of possible targets for the missile. The Fire button will launch the selected missile. For more information, see Firing a Missile.

To the left of the Fire button are two additional buttons. The Freeze/Resume button toggles the freeze state for the scenario and has the same effect as clicking either the Freeze Exercise or Run Exercise button on the main application toolbar. The Reset button has the same effect as the Reset Scenario button on the main application toolbar.

On the right side of the window is a drop-down list allowing the user to change between the following control modes for the entity:

  • Fly
    • This mode causes the entity to exhibit simple airplane-like behavior. The joystick's Left/Right axis controls the entity's roll. Pulling back on the joystick pitches the entity nose-up, while pushing forward on the joystick pitches the entity nose-down.
  • Magic Carpet
    • This mode causes the entity to move along a plane parallel to the ground. The joystick's Left/Right axis moves the entity left and right. The Up/Down axis moves the entity forward and backward.
  • Roll/Pitch
    • This mode allows the Left/Right and Up/Down axes of the joystick to control the entity's roll and pitch, respectively.
  • Yaw/Z
    • This mode allows the Left/Right and Up/Down axes of the joystick to control the entity's yaw and altitude, respecitvely.

The Increased Precision check box changes the sensitivity of the simulated joystick or joystick peripheral. Checking this box will allow the user to "fine tune" the position and attitude of an entity.

The Airspeed control changes the speed of the entity when it is in the Fly control mode. The range of this control is from –300 to 1200 knots. A negative airspeed value causes the entity to fly backward.

Note: For rotorcraft entities (specified by the ROTORCRAFT subclass in the ENTITIES.DEF file), the Airspeed control is replaced with a Collective control ranging from 0 to 100%. A value of 50% causes the entity to maintain a constant altitude. Decreasing the value decreases the altitude; increasing the value increases the altitude.

Keyboard Flight

Select an entity and press Ctrl+K to enable or disable keyboard flight. The KBD FLY pane in the status bar is green while keyboard flight is enabled and has a faint red background while it is disabled. Double-clicking the status pane also toggles keyboard flight.

KeyFlight control
WPitch nose-down
SPitch nose-up
ARoll left
DRoll right
QYaw left
EYaw right
RIncrease airspeed or collective
FDecrease airspeed or collective

Keyboard flight is deliberately disabled when HEMU loses application focus or when the user begins interacting with another control, including an entity tree, edit field, drop-down list, check box, button, or slider. Press Ctrl+K again after completing the UI operation to resume keyboard flight. Hover over the KBD FLY status pane for a shortcut reminder.

Physical Joystick Throttle

A physical throttle uses soft takeover to avoid an abrupt airspeed or collective jump. Move the hardware throttle close to the entity's current value. The status bar reports when the values match, and the Fly window's airspeed or collective slider turns green while the hardware throttle owns the value. A mouse, keyboard, or other manual speed change releases hardware throttle control until the values match again. The full physical throttle travel controls 0 to 1200 knots, or 0 to 100 percent collective for a rotorcraft. Physical joystick, POV hat, and keyboard throttle input cannot command a negative speed.

Joystick buttons fire the selected weapon, select the next target, freeze or resume the exercise, and cycle input precision. They do not change the entity's flight mode. Select Fly, Magic Carpet, Roll/Pitch, or Yaw/Z from this window's mode list. See Using a Joystick Peripheral for the complete button mapping.

In the lower right-hand corner of the Fly window are text areas showing the entity's current position and attitude.
