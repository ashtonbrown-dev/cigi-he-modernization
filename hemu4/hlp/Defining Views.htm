CIGI Host Emulator 3

Defining Views

The VIEWS.DEF file is used to define views and configure viewport and viewing volume attributes on the CIGI Host Emulator. The following is an example view definition:

VIEW
{
	NAME = "Front";
	ID = 0;
	GROUP = 1;
	TYPE = 0;
	PROJECTION_TYPE = PERSPECTIVE;
	REPLICATION_MODE = NONE;
	MIRROR_MODE = NONE;
	X_OFFSET = 0;
	Y_OFFSET = 0;
	Z_OFFSET = 0;
	YAW = 0;
	PITCH = 0;
	ROLL = 0;
	TOP = 30;
	BOTTOM = -30;
	LEFT = -40;
	RIGHT = 40;
	NEAR = 1;
	FAR = 100000;

	COMPONENT
	{
		NAME        = "Canopy Bow Mask";
		ID          = 4002;
		DEF_STATE   = 1;

		STATE
		{
			NAME  = "Off";
			VALUE = 0;
		}

		STATE
		{
			NAME  = "On";
			VALUE = 1;
		}
	}
}

VIEW definitions define the following parameters:

  • NAME
    A descriptive, user-recognizable name for the view. The name must be enclosed within double quotation marks ("…").
  • ID
    An integer that specifies the view.
  • GROUP
    An integer from 1 through 255 that specifies the group to which the view belongs. A value of zero indicates that the view should not be grouped. See Defining View Groups for information on how to specify view group properties.
  • TYPE
    An integer from 0 through 7 corresponding to an application-specific view type.  See the CIGI 3 ICD for more information.
  • PROJECTION_TYPE
    Specifies the type of viewing volume for the view. Acceptable values are PERSPECTIVE and PARALLEL.
  • REPLICATION_MODE
    Specifies the pixel replication mode for the view. The following keywords are valid:

     
    Keyword       Description
    NONE No replication
    1X2 Vertical pixel replication (line-doubling)
    2X1 Horizontal pixel replication
    2X2 Vertical and horizontal pixel replication

  • MIRROR_MODE
    Specifies the mirroring function used for the view. The following keywords are valid:

     
    Keyword       Description
    NONE No mirroring
    HORIZONTAL Flip view horizontally
    VERTICAL Flip view vertically
    BOTH Flip view horizontally and vertically

  • X_OFFSET
    Specifies the default position in meters along the parent entity's X axis.
  • Y_OFFSET
    Specifies the default position in meters along the parent entity's Y axis.
  • Z_OFFSET
    Specifies the default position in meters along the parent entity's Z axis.
  • YAW
    Specifies the default yaw in degrees with respect to the parent entity's coordinate system.
  • PITCH
    Specifies the default pitch in degrees with respect to the parent entity's coordinate system.
  • ROLL
    Specifies the default roll in degrees with respect to the parent entity's coordinate system.
  • TOP
    Specifies the top half-angle (in degrees) of the viewing volume.
  • BOTTOM
    Specifies the bottom half-angle (in degrees) of the viewing volume.
  • LEFT
    Specifies the left half-angle (in degrees) of the viewing volume.
  • RIGHT
    Specifies the right half-angle (in degrees) of the viewing volume.
  • NEAR
    Specifies the near clipping distance (in meters) for the view.
  • FAR
    Specifies the far clipping distance (in meters) for the view.
  • COMPONENT
    Defines a component for the view.  Discrete component definitions specify the following parameters:
    • NAME
      A descriptive, user-recognizable name for the component. The name must be enclosed within double quotation marks ("…").
    • ID
      An integer that uniquely identifies the component for a given view.
    • DEF_STATE
      An integer that specifies the default discrete state for the component.
    • DEF_DATA1
      An optional pair of parameters that specify the default value and datum type for the 1st user-defined datum of the component. The value may be an integer or floating-point value. The datum type is specified by one of the following keywords:

       
      Keyword Description
      INT32 32-bit signed integer
      UINT32 32-bit unsigned integer
      INT64 64-bit signed integer
      UINT64 64-bit unsigned integer
      FLOAT 32-bit single-precision floating-point number
      DOUBLE 64-bit double-precision floating-point number

      The value must precede the datum type and the two must be separated by a comma.
    • DEF_DATA2
      An optional pair of parameters that specify the default value and datum type for the 2nd user-defined datum of the component. The parameters are used as described by DEF_DATA1. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA3
      An optional pair of parameters that specify the default value and datum type for the 3rd user-defined datum of the component. The parameters are used as described by DEF_DATA2. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA4
      An optional pair of parameters that specify the default value and datum type for the 4th user-defined datum of the component. The parameters are used as described by DEF_DATA3. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA5
      An optional pair of parameters that specify the default value and datum type for the 5th user-defined datum of the component. The parameters are used as described by DEF_DATA4. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA6
      An optional pair of parameters that specify the default value and datum type for the 6th user-defined datum of the component. The parameters are used as described by DEF_DATA1; however, the datum type must be a 32-bit type. This parameter pair is not used if DEF_DATA5 specifies a 64-bit data type.
    • STATE
      Defines a discrete state for the component. Each STATE block must include the following parameters:
      • NAME
        A descriptive, user-recognizable name for the component state. The name must be enclosed within double quotation marks ("…").
      • VALUE
        An integer that specifies the value associated with the state.

      The parameters of the STATE block must be contained within a single set of brackets ({...}).
