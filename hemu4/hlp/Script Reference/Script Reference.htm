CIGI Host Emulator 3

Script Reference

The CIGI Host Emulator contains a script facility that may be used to perform a series of prearranged actions and to send a sequence of CIGI packets to the Image Generator. Scripts may be used to perform a tedious or complicated series of actions in a single step. They may be used to perform recursion tests when the results must be reproducible. They may also be used in lieu of a scenario (.sf3) file when the definition (.def) files are expected to be changed.

Each action is initiated by a script command. A command consists of a keyword followed by zero or more parameters. All parameters listed for any given command in this help file are mandatory. Each command must be on its own line, and each command and its parameters must be on the same line. No line may be longer than 255 characters. Commands are processed in the order in which they are listed in the script. Keywords are not case-sensitive.

Comments may be used to annotate a script. A comment may appear on its own line, or it may follow a command. Each comment must begin with a pound sign (#). Each line of a multi-line comment must begin with a pound sign.
