CIGI Host Emulator 2.1

Creating a New Entity

To create a new entity, click the Add Entity button on the Entity Tree toolbar. A new entity will appear in the entity tree as shown below.

New Entity

The new entity will be named "New Entity" and will be assigned the next available entity ID. If one or more entities have been deleted, the entity will be assigned a recycled ID in the order in which the entities were deleted. Otherwise, the next unused ID will be assigned.

Because some IGs might not be able to allow an entity's type to spontaneously change without first deleting the entity, the CIGI Host Emulator makes no assumptions about the entity type. The type of the new entity, therefore, is undefined. This is indicated by the question mark icon in the entity tree as shown in the figure above. The user must select an entity type from the Type list as shown:

Selecting an Entity Type

When the user selects an entity type, the entity's name and icon in the tree are updated.

When the Ownship is created, it is placed at the default position specified for the database in the DATABASES.DEF file. If no default position is specified, the Ownship is placed at the database origin specified in the file. Successive entities are placed in a 60-meter-diameter circle around the Ownship at intervals of approximately 45 degrees.
