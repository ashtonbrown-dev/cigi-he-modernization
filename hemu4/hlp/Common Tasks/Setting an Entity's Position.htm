Setting an Entity's Position

Every entity, whether it be a basic entity, special effect, or weather phenomenon, may be positioned relative to its reference coordinate system. The method for setting an entity's position depends on whether the entity is attached to a parent.

Top-Level (unattached) Entities:

For top-level entities (those entities whose Parent field is set to "None"), coordinates are specified in the world (Geodetic) coordinate system. The position may be set by entering the desired Latitude, Longitude, and Altitude values.

The position may also be set by selecting an entity from the Relative to list and specifying the Bearing, Range, and Altitude. The Latitude and Longitude values will be automatically calculated.

Child Entites:

For child entities, coordinates are specified in the parent's body coordinate system. The position may be set by entering an offset from the parent entity along the its X, Y, and Z axes. These correspond to the X Offset, Y Offset, and Z Offset fields, respectively.

If the parent entity is configured with attach points in the ENTITIES.DEF file, the attach points will be listed in the Attach to list. Selecting an attach point will cause the X Offset, Y Offset, Z Offset, Heading, Pitch, and Roll fields to be filled in automatically. Selecting "Arbitrary" from the list will dissociate the entity from its assigned attach point.
