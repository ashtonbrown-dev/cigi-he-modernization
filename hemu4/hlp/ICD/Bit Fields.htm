Bit Fields

To reduce packet size and Ethernet traffic, Booleans, enumerated values, and integers with small ranges typically use only the bits they need. Memory may be divided into bit fields to allow the data to be "packed" into as small a space as possible. Example of Multiple Bit Fields

The above word can be broken down as follows:

  • Val1 is a 4-bit unsigned integer value with a range of 0 through 24.

  • Val2 is a 3-bit unsigned integer value with a range of 0 through 23.

  • V3 is a single-bit binary value with possible values 0 and 1.

  • V4 is a single-bit binary value with possible values 0 and 1.

  • Val5 is a 2-bit unsigned integer value with a range of 0 through 22.

  • The bits marked as “Reserved” are unused and must contain zero.

The remaining two bytes can be used as two 8-bit integers or one 16-bit integer (signed or unsigned).
