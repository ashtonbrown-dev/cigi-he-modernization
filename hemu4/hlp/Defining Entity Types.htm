Editing ENTITIES.DEF

The ENTITIES.DEF file specifies the properties of each entity type. A typical entity definition is shown below. An example definition for a weather entity may be found in the Defining Weather Entity Types topic.

entity
{
   name = "F15-C";
   type = 103;
   class = fixedwing;
   alpha = 255;
   inherit_alpha = false;
   clamp_mode = none;
   anim_direction = forward;
   anim_loop = false;
 
   // The ARTPART definition specifies the name, ID, and
   // default position and orientation of an articulated part
   // contained within a model.
   artpart
   {
      name = "Slats";
      id = 11;
      position = 0, 0, 0;			// x, y, z
      position_enable = false, false, false;	// x, y, z
      rotation = 0, 1.5, 0;			// y, p, r
      rotation_enable = false, true, false;	// y, p, r
   }
 
   // The COMPONENT definition specifies the name, ID, and
   // valid states for a discrete component contained within a
   // model.
   component
   {
      name = "Speed Brake";
      id = 1002;
      def_state = 0;             // The default state
      def_data1 = 100, INT32;    // 1st word is 32-bit integer = 100
      def_data2 = 100, FLOAT;    // 2nd word is 32-bit float = 100.0
      def_data3 = 123, INT64;    // 3rd & 4th words 64-bit integer
      def_data5 = 123.0, DOUBLE; // 5th & 6th words 64-bit float
      
      // def_data4 and def_data6 are not used in this case because
      // those words are used with def_data3 and def_data5,
      // respectively, to make 64-bit datatypes.
 
      state
      {
         name = "Down";
         value = 0;
      }
 
      state
      {
         name = "Up";
         value = 1;
      }
   }
 
   // The COLLISION_SEGMENT definition specifies the ID and endpoints
   // of a segment to be automatically created for each instance of
   // the given type.
   collision_segment
   {
      name = "Fore-to-Aft";
      id = 0;
      enabled = true;
      start = 5.2, 0, 0;	// x, y, z offset
      end = -10.3, 0, 0;	// x, y, z offset
      material_mask = FFFF;
   }
 
   // The COLLISION_VOLUME definition specifies the ID and bounds
   // of a volume to be automatically created for each instance of
   // the given type.
   collision_volume
   {
      name = "Fuselage";
      id = 0;
      type = CUBOID;
      enabled = true;
      position = -8, 0, 0;	// x, y, z offset
      rotation = 0, 0, 0;	// yaw, pitch, roll
      width = 10.0;
      height = 5.0;
      depth = 15.5;
   }
 
   // The ATTACH_POINT definition is used by the Host Emulator
   // to specify attach points for child entities such as missiles.
   attach_point
   {
      id = 0;
      name = "Left Wing Pylon";
      position = 5, 0, 2;
      rotation = 0, 0, 0;
   }
}

ENTITY definitions specify the following parameters:

  • NAME
    A descriptive, user-recognizable name for the entity type. This name is used to populate lists and combo boxes in the CIGI Host Emulator's graphical user interface. The name must be enclosed within double quotation marks ("…").
  • TYPE
    An integer that uniquely identifies the entity type. This value must correspond to the entity model type value specified in the IG configuration.
  • CLASS

    A general class designation for the entity type. This parameter indicates to the CIGI Host Emulator which physics model, icon, and units of measure to use for each entity. Supported subtypes are specified by the following keywords:

     
    Keyword Icon Description
    FIXEDWING Fast-jet and other airplanes
    ROTORCRAFT Helicopters
    WATERCRAFT Ships and boats
    GROUNDMIL Tanks, transports, and other military ground vehicles
    GROUNDCIV Automobiles and other civilian vehicles
    MISSILE Missiles
    BOMB Bombs
    CULTURE Buildings, bridges, and other cultural models
    WEATHER Entity-based weather phenomena
    ANIMATION Animation entities
    TESTPATTERN Video test pattern models
    GENERIC All other entities

    These keywords must not be enclosed within quotes.

  • ALPHA
    An integer from 0 through 255 specifying the default alpha value for the entity type.
  • INHERIT_ALPHA
    A Boolean value that specifies whether entities of this type inherit the alpha values of their parents by default.  Acceptable values are TRUE or FALSE, YES or NO, and 1 or 0.
  • CLAMP_MODE
    The default clamp mode for the entity type. The mode may be represented by one of the following keywords:

     
    Keyword Description
    NONE The entity is not clamped
    CONFORMAL The entity is clamped and its pitch and roll conform to the terrain
    NONCONFORMAL The entity is clamped and its pitch and roll are independent of the terrain

  • ANIMATION_DIRECTION
    A value that specifies the default direction of an animation.  Acceptable values are FORWARD or BACKWARD. This parameter is used only when CLASS is set to ANIMATION.
  • ANIMATION_LOOP
    A  Boolean value that specifies whether entities of this type inherit the alpha values of their parents by default.  Acceptable values are TRUE or FALSE, YES or NO, and 1 or 0. This parameter is used only when CLASS is set to ANIMATION.
  • ARTPART
    Defines an articulated part for entities of the given type.  An articulated part is a part of a model, such as an aircraft's landing gear, that can be moved and/or rotated relative to the rest of the model.  Articulated part definitions may contain the following parameters:
    • NAME
      A descriptive, user-recognizable name for the part. The name must be enclosed within double quotation marks ("…").
    • ID
      An integer that uniquely identifies the part for a given entity. This number must match the IG's configuration so that it can associate this ID with a named transformation node within the entity's model.
    • OFFSET
      A set of three floating-point numbers, separated by commas, that specify the default X, Y, and Z positions of the part, respectively.  These positions are specified in meters from the entity's reference point.
    • ROTATION
      A set of three floating-point numbers, separated by commas, that specify the default yaw, pitch, and roll of the part, respectively.  The orientation is specified in degrees relative to the entity's body coordinate system.
  • COMPONENT
    Defines a component for entities of the given type.  Discrete component definitions specify the following parameters:
    • NAME
      A descriptive, user-recognizable name for the component. The name must be enclosed within double quotation marks ("…").
    • ID
      An integer that uniquely identifies the component for entities of a given type.
    • DEF_STATE
      An integer that specifies the default discrete state for the component.
    • DEF_DATA1
      An optional pair of parameters that specify the default value and datum type for the 1st user-defined datum of the component. The value may be an integer or floating-point value. The datum type is specified by one of the following keywords:

       
      Keyword Description
      INT32 32-bit signed integer
      UINT32 32-bit unsigned integer
      INT64 64-bit signed integer
      UINT64 64-bit unsigned integer
      FLOAT 32-bit single-precision floating-point number
      DOUBLE 64-bit double-precision floating-point number

      The value must precede the datum type and the two must be separated by a comma.
    • DEF_DATA2
      An optional pair of parameters that specify the default value and datum type for the 2nd user-defined datum of the component. The parameters are used as described by DEF_DATA1. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA3
      An optional pair of parameters that specify the default value and datum type for the 3rd user-defined datum of the component. The parameters are used as described by DEF_DATA2. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA4
      An optional pair of parameters that specify the default value and datum type for the 4th user-defined datum of the component. The parameters are used as described by DEF_DATA3. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA5
      An optional pair of parameters that specify the default value and datum type for the 5th user-defined datum of the component. The parameters are used as described by DEF_DATA4. This parameter pair is not used if DEF_DATA1 specifies a 64-bit data type.
    • DEF_DATA6
      An optional pair of parameters that specify the default value and datum type for the 6th user-defined datum of the component. The parameters are used as described by DEF_DATA1; however, the datum type must be a 32-bit type. This parameter pair is not used if DEF_DATA5 specifies a 64-bit data type.
    • STATE
      Defines a discrete state for the component. Each STATE block must include the following parameters:
      • NAME
        A descriptive, user-recognizable name for the component state. The name must be enclosed within double quotation marks ("…").
      • VALUE
        An integer that specifies the value associated with the state.

      The parameters of the STATE block must be contained within a single set of brackets ({...}).

    Note:    Articulated parts are independent of discrete components.  In other words, an entity of a given type may have an articulated part and a discrete component with the same ID.  Further, an entity of a different type may also have an articulated part and/or a discrete component with that ID.

  • COLLISION_SEGMENT
    Defines a collision detection segment for entities of the given type. A collision detection segment is used by the IG to determine when a collision occurs with another entity or a part of the environment. Collision detection segment definitions may contain the following parameters:
    • ID
      An integer that uniquely identifies the collision detection segment for a given entity.
    • ENABLED
      A Boolean value that specifies whether the segment is enabled by default.  Acceptable values are TRUE or FALSE, YES or NO, and 1 or 0.
    • START
      A set of three floating-point numbers, separated by commas, that specify the relative X, Y, and Z positions of the origination endpoint, respectively. These positions are specified in meters from the entity's reference point.
    • END
      A set of three floating-point numbers, separated by commas, that specify the relative X, Y, and Z positions of the destination endpoint, respectively. These positions are specified in meters from the entity's reference point.
    • MATERIAL_MASK
      A 32-bit hexadecimal value that is used to exclude certain types of surfaces from the intersection calculations. Refer to the CIGI Interface Control Document or the CIGI 3 Interface Control Document for more information.
  • COLLISION_VOLUME
    Defines a collision detection volume for entities of the given type. A collision detection volume is used by the IG to determine when a collision occurs with another collision volume. Collision detection volume definitions specify the following parameters:
    • ID
      An integer that uniquely identifies the collision detection swept volume for a given entity.
    • TYPE

      The shape of the volume. Valid keywords are SPHERE and CUBOID.

    • ENABLED
      A Boolean value that specifies whether the volume is enabled by default.  Acceptable values are TRUE or FALSE, YES or NO, and 1 or 0.
    • CENTER
      A set of three floating-point numbers, separated by commas, that specify the relative X, Y, and Z positions of the center of the volume, respectively. These positions are specified in meters from the entity's reference point.
    • DEPTH
      A floating-point number that specifies the size of the volume, in meters, along the entity's X axis. This parameter used only when TYPE is set to CUBOID.
    • WIDTH
      A floating-point number that specifies the size of the volume, in meters, along the entity's Y axis. This parameter used only when TYPE is set to CUBOID.
    • HEIGHT
      A floating-point number that specifies the size of the volume, in meters, along the entity's Z axis. This parameter used only when TYPE is set to CUBOID.
    • RADIUS

      A floating-point number that specifies the radius of the volume in meters. This parameter is used only when TYPE is set to SPHERE.

    Note:    Collision detection segments are independent of collision detection volumes.  In other words, an entity of a given type may have a segment and a volume with the same ID.  Further, an entity of a different type may also have an segment and/or a volume with that ID.

  • ATTACH_POINT
    Defines an attach point for entities of the given type. An attach point defines a "slot," or relative position and orientation, at which child entities may be easily attached. Attach point definitions specify the following parameters:
    • NAME
      A descriptive, user-recognizable name for the point. The name must be enclosed within double quotation marks ("…").
    • ID
      An integer that uniquely identifies the attach point for a given entity.
    • OFFSET
      A set of three floating-point numbers, separated by commas, that specify the relative X, Y, and Z positions of the attached child, respectively. These positions are specified in meters from the parent entity's reference point.
    • ROTATION
      A set of three floating-point numbers, separated by commas, that specify the relative yaw, pitch, and roll of the attached child, respectively. The orientation is specified in degrees relative to the parent entity's body coordinate system.
