Embedded External Tools

HEMU can launch a separate Windows application and host its main window inside a tab beside the Messages and Capture tabs. The tool remains a separate process: HEMU does not link its source code or libraries.

Configuration Files

External tabs are configured relative to the directory containing Hemu4.exe:

  • config\external_tabs.ini contains safe, distributable defaults.
  • config\external_tabs.user.ini contains machine-specific or private tools. When present, this file replaces the default file.

The StartupTab value selects the initial tab. Each ExternalTabN section supports Enabled, Title, Executable, WorkingDirectory, Arguments, LaunchOnSelect, and CloseWithHemu. Relative executable and working-directory paths are resolved from the HEMU application directory.

[ExternalTabs]
Count=1
StartupTab=ADI

[ExternalTab0]
Enabled=1
Title=ADI
Executable=..\..\..\tools\cigi-adi\dist\cigi-adi\cigi-adi.exe
WorkingDirectory=..\..\..\tools\cigi-adi\dist\cigi-adi
Arguments=--embedded --cigi --listen 0.0.0.0:8014 --reply 127.0.0.1:8015
LaunchOnSelect=1
CloseWithHemu=1

Window-Embedding Contract

HEMU starts the process hidden, finds an unowned top-level window created by that process, changes it into a child window, sizes it to the tab, and reveals it without requesting activation. Hidden top-level windows are valid candidates, so a cooperative tool does not need to flash a desktop window before attachment.

A tool intended for hosting should provide an option that creates its native window hidden and initially unfocused. It should not call SetForegroundWindow or otherwise activate itself during startup. HEMU passes --embedded to CIGI ADI for this purpose. Standalone ADI behavior is unchanged when the option is omitted.

After attachment, HEMU restores its own keyboard focus only if HEMU or the newly launched tool still owned the foreground. It does not take focus back if the user selected another application while the tool was starting.

Keyboard Focus

An embedded tool still belongs to a different process and UI thread. Clicking inside its tab can therefore give keyboard focus to that tool. Click a HEMU control before using HEMU keyboard commands such as Ctrl+K. The hidden and unfocused startup contract prevents the tool from taking focus merely because HEMU launched it.

Process Lifetime

When CloseWithHemu=1, HEMU contains the external process in a kill-on-close Windows Job Object and closes it with HEMU. When set to zero, HEMU detaches and restores the external window when the host page closes.
