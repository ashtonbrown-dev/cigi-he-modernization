Version 3.3.2
-------------------------------------------------------------------------
- Audited CIGI packet implementations for correctness.
    - Fixed incorrect field sizes in CIGI_POSITION_REQUEST and others.
    - Implemented missing "Request ID" field in Environmental Conditions packet.
    - Other misc. fixes.

- Fixed buggy behavior in Component States pane (on Entity page). It was often
  the case that when trying to set a state in the pane, you had to set it to a
  different state first. (For example, if you wanted to turn a switch ON, just
  selecting ON would not always work. You had to select OFF first, and then ON.)
  
- An IG Control packet with bogus data that was sent from the Host Emulator
  during initial connection to an IG should no longer occur.

- Fixed incorrect positioning of entities on terrain database [re]load.

- Implemented CIGI 3.3 versions of Component Control and Short Component Control.

- Numeric fields that show various state (e.g. entity state like lat/long) are
  now grayed out and uneditable while in Run mode. Putting the HE into Freeze
  will ungray the fields for editing.
    - The HE used to function this way, but the ability to edit state while
      in Run was added. However, the feature was proving to be problematic
      and not easily solvable with the current GUI design. It was decided
      to revert to the old behavior.

- Fixed Component-related bugs that were introduced in 3.3.1.
    - Terrain/Environment-related components will now show up in the correct lists.
    - Component States will again be listed correctly for components with instance
      IDs set to -1 in .def files.

- Menu items for symbology-related packets (e.g. Actions -> Send Packet) are
  now disabled when the user configures the Host Emulator to use CIGI versions
  below 3.3 in the Setup screen. Symbology packets will also be disabled in
  scripts when the version is too low.

- Capture filtering checkboxes for symbology-related packets now work and persist.

- Scripting:
    - Fixed GTERRAIN_COMPONENT commands not looking for database terrain
      components, causing "invalid component" errors when running scripts.
    - Add "instance ID" argument to documentation for various XXX_COMPONENT commands.
    - Support for scripting System Components should now be fully in place.

- Fly window:
    - Switching fly modes will now reset the joystick to the center.
    - The current fly mode will be maintained when hitting Reset.
    - Made the horizontal slider in the Fly window slightly wider, so that it's
      easier (and possible) to manually set the slider back to 0 by hand.
    - The displayed Airspeed/Speed number is now rounded, so that it can
      actually top out at e.g. 1200 instead of 1199.
    - The window can now be closed by pressing Escape.
    - The Freeze/Resume button will no longer erroneously say "Freeze" on
      application start (when no exercise is being run).

- Scripts dialog:
    - The width of the scripts dropdown list is now dynamically sized to match
      the width of the longest path name, so that the paths do not get cut off.
    - The vertical allowance has also been increased, so that more of the list
      can be shown before the vertical scroll bar kicks in.

- Various other user interface fixes/improvements:
    - Fixed the odd GUI behavior of the Time field in the Celestial Sphere
      section of Environment -> Sky and Global Atmosphere.
    - Script commands that set entity position will now cause a refresh in the GUI
      entity state view, so that the user can see the updated entity position.
    - All windows/dialogs are now centered upon first opening them.
    - The list of packets under "Send Packet" has been alphabetized.
    - Fixed misnamed Line of Sight Vector packet ("Vector" was misnamed "Volume".)
    - Multiple packets are no longer sent when using Celestial Sphere Control dialog.
    - On the HAT/HOT and LOS tabs in the Entity pane, "Request ID Range" entries
      now correctly show a range that spans 64 IDs (instead of 16).
    - Fixed the Enable Segment/Volume checkbox on the Collision Detection tab.

- DummyIG now also prints:
    - Atmosphere Control Packet
    - Celestial Sphere Control
    - Collision Detection Segment Definition
    - Collision Detection Volume Definition
    - Earth Reference Model Definition
    - Environmental Condition
    - Environmental Region Control
    - Maritime Surface Conditions Control
    - Motion Tracker Control
    - Position Request
    - Rate Control
    - Sensor Control
    - Terrestrial Surface Conditions Control
    - Trajectory Definition
    - Wave Control

- Added the ability to have multiple terrain components with the same
  component ID but different instance IDs.

Version 3.2.0
-------------------------------------------------------------------------
- Added support for CIGI 3.2 features:
	- Added CIGI Version combo to Setup dialog
	- Modified Start of Frame and IG Control packets
	- Added Coord System parameter to Rate Control packet
	- Modified HAT/HOT Request, LOS Segment Request, and LOS Vector

  Part Control, and Weather Control packets for Big-Endian emulation.

  after the main process exited.

- Added the following script commands:
	ATMOSPHERE_COMPONENT_END_UPDATE
	CELSPHERE_COMPONENT_END_UPDATE
	ENTITY_COMPONENT_END_UPDATE

	SYSTEM_COMPONENT_END_UPDATE
	VIEW_COMPONENT_END_UPDATE
	VIEWGROUP_COMPONENT_END_UPDATE

- Improved the reliability of the FRAME and FRAME_WAIT script commands.

- Fixed a bug in the driver preventing packet recording from working
  after the first time.

- Fixed problem with date/time not being updated correctly from
  Sky and Global Atmosphere properties on Environment tab.

- Fixed bug where view groups' pitch and roll values were not being set
  correctly.

- Fixed LINEAR_RATE and ANGULAR_RATE script keywords so that one does
  not override (zero out) the other.

- Fixed Big-Endian setting in Setup dialog not being saved in the Windows
  registry.

- Cleaned up driver thread shutdown events.  This should (hopefully) fix
  situations where the driver does not terminate properly.


Version 3.1.3
-------------------------------------------------------------------------
- Replaced 3.1.0/3.1.1 data-sharing mechanism. This should fix MANY bugs.

- Fixed annoying problem where deleting the contents of an entity field
  would cause an error window to pop up repeatedly.

- Added VIEW_ATTACH and VIEWGROUP_ATTACH keywords as documented in the
  help file. The commands were actually implemented as ATTACH_VIEW and
  ATTACH_VIEWGROUP; those commands are still implemented but not
  documented.

- Fixed the PAUSE_MESSAGE and CLEAR_MESSAGES script commands.

- Fixed Celestial Sphere Control and Weather Control CIGI packets not
  being sent.

- Fixed occurrence of two Entity Control packets when assigning a parent
  to a child entity. This was a quick fix and is not guaranteed to stop
  every occurrence; however, providing a proper fix will require more
  effort than it is worth. This is related to a problem where an Entity
  Control packet is sent when an entity's fly mode is changed.

- Fixed problem where changing the Sky and Global Atmosphere properties
  in the Environment tab did not cause packets to be sent or generated
  erroneous packets. This applied to both celestial sphere and global
  atmosphere properties.

- Added a Fly Mode indicator to the Attributes area of the entity state
  window. Rearranged some of the items to accommodate the addition.

- Added multi-type combo boxes to Send Short Component Control Packet
  dialog.

- Fixed Short Component Control packets not being sent if two or fewer
  user-data fields were needed.

- Fixed ENTITY_PITCH_RELATIVE script command.

- Fixed Entity Controls being sent when airspeed, fly mode, and other
  non-cigi data were changed.

- Added range circle labels to the situation display.

- Message window now gives the coordinates of the next waypoint when a
  script is running.

- Decreased joystick's POV hat sensitivity for Magic Carpet mode.

- Fixed playback loop mode.
